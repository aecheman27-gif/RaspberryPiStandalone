What this gets you

Follow these phases top to bottom and you'll go from a blank microSD card to writing Python in PyCharm on Windows that runs on the Pi and blinks a real LED. Tick each box as you go — your progress is saved in this browser automatically.

⚡ Fast start (already have the Pi imaged?)

  1. Double-click setup-pi-connection.bat — installs SSH key on the Pi, makes ssh edith work, creates ~/PiCourse, downloads TigerVNC. (Type the Pi password once.)
  2. SSH in and run pi-classroom-setup.sh — installs all kit libraries + enables I2C/SPI, then sudo reboot.
  3. Double-click windows-classroom-tools.bat — installs PyCharm + Git, and prints the exact PyCharm-to-Pi steps.
  4. Wire the LED (Phase 7), then scp Blink.py edith:~/PiCourse/ and ssh -t edith "python3 ~/PiCourse/Blink.py". 💡

The phases below explain every step and the gotchas we hit, with checkboxes.

1

Flash the SD card (headless)

Raspberry Pi Imager
Install Raspberry Pi Imager and choose Pi 5 + Raspberry Pi OS (64-bit, Desktop)
From raspberrypi.com/software. Pick your microSD as the storage.
In “Edit Settings”: hostname edith, user aecheman + password, Wi-Fi SSID/password, country US
This is what makes it work with no monitor/keyboard. Full detail is in Pi5.md.
Enable SSH (Services tab), write the card, boot the Pi, wait 2–4 min
2

Connect from Windows (one script)

SSH key
Double-click setup-pi-connection.bat
Generates your SSH key, adds the ssh edith shortcut, installs the key on the Pi, creates ~/PiCourse, and downloads TigerVNC.
Type your Pi password once when it asks — that's the key install. After this, SSH never asks again.
Verify passwordless login
ssh edith "hostname"
Should print edith with no password prompt. That's the signal everything downstream will work.
3

Reach it from anywhere (Tailscale)

optional but recommended
Install Tailscale on the Pi and sign in (use GitHub or your usual account)
ssh edith curl -fsSL https://tailscale.com/install.sh | sh sudo tailscale up
Open the printed URL in your browser to authorize. The Pi joins your tailnet as edith (IP 100.104.213.4).
In the Tailscale admin console, Disable key expiry for edith
login.tailscale.com/admin/machines → edith → ⋯ → Disable key expiry. Keeps an always-on device from dropping off.
4

Prep the Pi for the kit

libraries + I2C/SPI
Copy pi-classroom-setup.sh to the Pi and run it
scp pi-classroom-setup.sh edith:~/ ssh -t edith "bash ~/pi-classroom-setup.sh"
Installs gpiozero, smbus, spidev, i2c-tools, Thonny; enables I2C & SPI (needed for the LCD/ADC/sensor projects); clones the Freenove code; creates ~/PiCourse.
Reboot so I2C/SPI take effect
ssh -t edith "sudo reboot"
Use ssh -t for any sudo command — sudo needs a terminal to read the password.
5

Graphical desktop (VNC)

TigerVNC
Enable the built-in VNC server on the Pi
ssh -t edith "sudo raspi-config"
Interface Options → VNC → Yes. Also System Options → Boot/Auto Login → Desktop Autologin.
Connect with TigerVNC (already downloaded by the script in Phase 2)
Run %USERPROFILE%\PiTools\Connect-edith-VNC.bat, or open the viewer and enter 100.104.213.4:5900.
Use TigerVNC, not RealVNC — RealVNC now charges to connect to non-RealVNC servers. On first connect, accept the two self-signed-certificate prompts (safe — it's your Pi over an encrypted link).
6

PyCharm → run code on the Pi

the fiddly part — read the warnings
Install the unified PyCharm and activate Pro
Run windows-classroom-tools.bat, or get it from jetbrains.com/pycharm/download (the “Unified Product”). Then Help → Register → start the trial or activate a free student/.edu license.
Plain Community Edition can't do remote-SSH — its Add-Interpreter menu only shows “Add Local Interpreter.” You need the unified/Pro one.
New project on the C: drive
Don't put the project (or its venv) on an external/F: drive — Windows denies write access and the venv creation fails.
Settings → Python → Add Interpreter → On SSH → Existing
Host 100.104.213.4, user aecheman, Auth = Key pair → C:\Users\aeche\.ssh\id_ed25519.
Environment = Select existing, Type = Python, Path = /usr/bin/python3
Do not “Generate new” virtualenv — the kit's libraries (gpiozero, smbus) are installed system-wide on the Pi and a fresh venv won't see them.
Point the upload folder at /home/aecheman/PiCourse
Settings → Build, Execution, Deployment → Deployment → your server → Connection tab → Root path = /home/aecheman/PiCourse; Mappings tab → Deployment path = /. Then menu → Tools → Deployment → Automatic Upload (always).
The default /tmp/pycharm_project_… gets wiped on every reboot and caused our “No such file / Permission denied” errors. Using your home folder fixes it for good.
Don't “update” the packages PyCharm lists for the interpreter
Those are managed by the Pi's apt. Updating them via pip can break Raspberry Pi OS. To update, use sudo apt full-upgrade on the Pi instead.
7

Wire the Blink LED

3 parts
Remove any chips from the breadboard (not needed for Blink) and seat the GPIO extension board + ribbon cable
Wire with the Pi powered off: ssh -t edith "sudo poweroff", wait for the green light to stop, unplug USB-C.
Place the 3 parts as shown: resistor, LED, jumper
LED long leg = the longer wire, goes toward the resistor/IO17 side. Backwards just means it won't light — no harm.
Plug USB-C back in, wait ~1 min, confirm it's back
ssh edith "hostname"
8

Run Blink 💡

the payoff
Create Blink.py with this code
from gpiozero import LED from time import sleep led = LED(17) while True: led.on() sleep(1) led.off() sleep(1)
Only the four lines under while True: are indented.
Run it — either way works
Command line (always works):
scp Blink.py edith:~/PiCourse/ ssh -t edith "python3 ~/PiCourse/Blink.py"
Or PyCharm: with the interpreter + deployment from Phase 6, just press the green ▶ Run. Stop with Ctrl+C / the red ■.
The LED blinks once per second 🎉
If nothing lights: LED is in backwards (swap its legs). If you get a Python/GPIO error, that actually means the code reached the hardware — check the pin and wiring.
